# Component preview screenshots

Drop one screenshot per component here. The `/components` landing reads them for
the featured 3D cards and the image-reveal marquee.

- **Filename:** `<component-id>.png` (the id must match the route, e.g. `/components/glitch-card` → `glitch-card.png`)
- **Format:** PNG or WebP (update the extension in `src/pages/Components.tsx` `previewSrc()` if you use WebP)
- **Recommended size:** ~720×450 (16:10), keep them light
- Missing files fall back to a monogram automatically — add them incrementally.
